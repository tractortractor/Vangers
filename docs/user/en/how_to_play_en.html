What do you need to play

  1. Download "data", "libraries" and "executables" folders from latest release on github.
  2. Extract and move all contents of those folders to folder with your current installation of Vangers. Replace if files already exists.
  3. Launch one of executables.

Applies to all builds

Mini-map legend:

  • Big arrow - vanger.
  • Small orange arrow - fish.
  • Small red arrow - Jablee.
  • Small green square - item.
  • Big square with yellow borders - larva.
  • Big green empty square - passive swarm.
  • Big orange empty square - active swarm.
  • Green dot - beeb.
  • Orange dot - bullet.

zRECORDER, replays

IMPORTANT! Backup saves and options before playing replays. Save file will be overwritten during playing of replay if replay creator saved the game during recording. Other settings may be changed in similar way. If you are using all_debug_cheats or vangers_debug builds then replay can also change "*.prm" files. Saves are stored in "savegame" folder. Options are stored in "game.dat" and "controls.dat" files.


Records player's action if command line argument "w" or "W" is specified. File where to write must be specified right after this argument. Example: "/wfile_where_to_save", "/Wother_file_where_to_save".

Loads player's action if command line argument "p" or "P" is specified. File where to read from must be specified right after this argument. Example: "/pfile_where_to_read", "/Pother_file_where_to_read". Any control action which is not mouse moving, releasing keyboard button or releasing mouse button will stop playback.


Replays should be played with same build of vangers which were used for recording.

It is unknown if replays are cross-platform, so it's a good idea to specify your OS while posting them.

Options which are ok to modify while recording: all camera options, autorun, deformable landscape. Modification of other options may mess up recording/playing.


RNG seeds, controls and some options are saved in replay file.

While recording with joystick, replay size will be several times bigger.

While "GamePad" mode is on the game will record joystick movement events only when axis changes direction.

Mouse movements are not recorded so game interface does not react to them while recording.

While recording it's possible to steal counselor's items through walls.

Controls for other features:

  • "ctrl"/"shift" + "t" - Messes up time. Makes it too slow/fast.
  • "ctrl" + "f" - Show fps and player's coordinates.

Basic builds' description

"Executables" folder contains several builds. There are 2 types of executable folders: minor and major. Major one contains all interesting or important ones. Minor one contains boring builds or builds which do not affect the game much.

Major builds

  • no_debug

    Just the base game without any changes.

  • aci_escave_debug

    Ability to add money and change mood of next dialog line.

    Controls:

    • "1" - change next speech mode to normal (0).
    • "2" - good (1).
    • "3" - bad (2).
    • "4" - exile (3).

    • "numpad +" - add 10 000 beebs.

  • aci_no_shop_animation

    Disables animations at counselor's screen.

    First line of dialog won't show up automatically.

    Player can't be exiled.

  • aci_skip_mainmenu

    Skip main menu initialization and starts new game instantly. Going back to menu also skips it and starts new game.

    Settings which are set from in-game menu are not remembered while starting the game this way.

  • aci_skip_shop

    When starting new game or loading, escave initialization is skipped and player's vanger goes to world instantly. While going back to escave it's initialised like in base game.

  • aci_skip_shop_and_mainmenu

    Combines aci_skip_mainmenu and aci_skip_shop.

  • aci_skip_shop_and_main_menu_and_load_on_start

    Same as aci_skip_shop_and_mainmenu but loads first save instead of starting new game. First slot is at the top in save/load menus.

  • diagen_test

    Loads dialog related variables from "data.lst" if present. Makes all variables non-updatable. See "DATA_UPDATABLE" below.

    Prints dialog name, dialog mood (speech mode) and dialog itself in the console.

    Controls:

    • "5" - Next dialog line is printed in the console and skipped in game.
    • "6" - Same as previous but done 24 times.
    • "7" - Get list of available questions.
    • "8" - All available questions are asked and answered in the console.
    • "9" - Game process stops and waits for console input. Input is saved but not used.
    • "0" - Use last received console input.

    Console commands:

    • "GO <escave name>" - Start dialog of specified escave. Names match is case-sensitive. Possible escave names are: Podish, Incubator, VigBoo, Lampasso, Ogorod, ZeePa, B-Zone, Spobs, Threall. "GO Ogorod", for example.
    • "FINISH" - Stop current dialog sequence.
    • "STATUS" - Prints world name, escave name, room name, counselor name and level. Range of values for the level is 0-2. Higher level - more respect for player.
    • "ASK <entry>" - Ask available question. "ASK Agent", for example. See "7" above.
    • "ESTATUS <level>" - Change eleepods' level to specified variable. It's the same level that is printed while invoking STATUS command. "ESTATUS 0", for example.
    • "BSTATUS <level>" - Same as ESTATUS but for beeboorats.
    • "ZSTATUS <level>" - Same as ESTATUS but for zeexs.
    • "SSTATUS <level>" - Same as ESTATUS but for spectators.
    • "SAVE" - Save dialog related data in "diagen_saveff.txt" file. Should be viewed with hex editor since values are in raw bytes.
    • "LOAD" - Load dialog related data from "diagen_loadff.txt" file.

    Next console commands are dealing with dialog variables.

    Each time dialog starts or updates through console commands, dialog variables are updated from global variables. There is filter array which is used to prevent manually set dialog variables from being updated.

    • "DATA <dialog data index> <new value>" - Set dialog data found by index to specified new value, make this value non-updatable, then perform "DATALIST" action. Both input values must be integers. See "DATALIST".
    • "DATALIST" - Save variables list in "data.lst" file. You can modify those variables with "DATA" and "DATALIST0" commands. visitCounter and comingCounter are not part of data list, so they are not modifiable with mentioned commands. Use "VISIT" to modify them.
    • "DATALIST0" - Set all variables in data list to 0 and make them non-updatable then perform "DATALIST" action.
    • "DATALIST_READABLE" - Save variables in readable form in "readable_data.lst". ID, name and updatability are specified for each variable. Cannot be used as "data.lst" to load variables in game.
    • "DATALIST0_READABLE" - Same as "DATALIST0" but saving in readable form in "readable_data.lst" file. Completely useless in practice.
    • "DATA_UPDATABLE <dialog data index> <updatable>" - Change updatability of specified variable. Value of updatable dialog variable will be always overwritten by global variable.
    • "DATA_UPDATABLE_ALL <updatable>" - Change updatability of all variables.
    • "MAP" - Prints dialogs map in console.

        • Legend:

      • space - EMPTY - Unoccupied space.
      • "D" - DORMANT - Currently uninvolved.
      • "U" - USED - Already used.
      • "W" - WAITING - Active, waiting to be checked.
      • "O" - OPENED - Active with access.
      • "E" - EXPLODED - Involved.
      • skipped while mapped - HALFLIFE - Waiting for end.
    • "VISIT <escave ID> <visitCounter> <comingCounter>" - Change visitCounter and comingCounter of specified escave to specified values. Escave ID is first symbol of name, case-sensitive. See "GO" above for full list of names. visitCounter and comingCounter values are never actually used.

    Notes:

    • Console of some versions of Windows is not displaying UTF-8 symbols properly, so Russian texts are unreadable. Redirect output to file in this case, example of console command: "vangers_diagen_test > vangers_diagen_output.txt".

    • Eleepods' ranks:

      • 0 - Not enough Eleection wins. Counselors tends to give vague answers.
      • 1 - Eleection reward 1. Leepky explains how P-Beam works. Some answers are changed to more informative ones.
      • 2 - Eleection reward 2. Leepky gives key to Glorx.

    • Beeboorats' ranks:

      • 0 - Slave.
      • 1 - Commissar.
      • 2 - Agent.

    • Zeexens' ranks:

      • 0 - Stranger, ready to become ProtoZeek.
      • 1 - Known vanger.
      • 2 - Just arrived ProtoZeek.

    • Parapheen is giving different missions depending on spectators' level:

      • 0 - Get unique mechos.
      • 1 - Visit Threall.
      • 2 - Repopulate the chain.
    • Scanning of inventory items is broken while talking at Threall's yellow circle. Only Mechanic Messiah is spotted properly. Caused by different handling of inventory in escave and outside.
    • Threall's level displayed in status is actually level of last visited bunch.

  • imove_mouse_object

    When hovering mouse over button it becomes selected.

    Coordinates of selected button are printed at right-bottom of the screen.

    Selected button can be moved around.

    Moving outside of the screen crashes the game if moved up-down.

    Controls:

    • "numpad 4" - Move left by 1.
    • "numpad 8" - Move up by 1.
    • "numpad 6" - Move right by 1.
    • "numpad 2" - Move down by 1.

    • "Home" - Move left by 10.
    • "Page Down" - Move up by 10.
    • "End" - Move right by 10.
    • "Page Up" - Move down by 10.

    • "s" - Write c/c++ macro definitions of current buttons' positions to "screen.txt".

    • "tab" - Move screen back and forth so there are no observable changes.
    • "[" - Move screen left.
    • "]" - Move screen right.

      • Will cause crash if used too much or moved to left while being on first screen.

  • save_binary_packed_script_and_iscript_debug

    Each time menu is loaded it is saved in binary "*.scb" file.

    Saves data about scripts loading in "iscreen\setup.dat" file. Rewrites this file each time new menu is loaded.

  • sicher_debug

    No borders at main worlds.

    Probably "SICHER" is nickname of game developer Alexander "Sicher" Kotliar.

    Flying, digging and swimming requires no special items.

    All controls are default:

    • "Up" - Move forward.
    • "Down" - Move backward.
    • "Shift" - Acceleration.
    • "x" - Handbrake.
    • "Left" - Turn wheels left.
    • "Right" - Turn wheels right.
    • "End" - Turn over left.
    • "Page up" - Turn over right.
    • "Insert" or "a" - Activate KiD.
    • "Page down" or "z" - Turn device up.
    • "Home" - Turn device down.

    There are 3 special switches. Once fish/Jablee switch is on there is no way to turn it off since almost all controls are blocked. Only relaunching the game will unblock game controls.

    Fish/Jablee test switch makes player's mechos and everything close to it, behave like fish/Jablee. No other special behavior. Player's vanger is just swimming/flying at one direction.

    In this version "stop" switch does nothing.

    Controls:

    • "g" - Activate fish test switch.
    • "h" - Activate Jablee test switch.
    • "b" - Activate stop switch.

  • unit_debug_view

    AI paths are seen.

    Ray shows facing direction of controlled vanger.

    Controls:

    • "Fire weapon 1" - print next hint message.

  • vangers_debug

    Saves some network and 3d object statistics in "*.lst" file.

    All items are added to escave shops right from the start.

    Saves data about tabutask events in "tabu_event" file.

    Saves specific data about game world in "stand.dmp" file at certain moments.

    Number of beebs is set to 799 999 each time escave is entered. This happens before all other changes, so after ruBeecation number of beebs is 600. Does not apply to Spobs.


    "*.prm" values debug menu allows you to modify and save "*.prm" variables while playing.

    Controls:

    • "f" - Opens "*.prm" values debug menu.
    • "Page Down" - Select next variable at selected line.
    • "Page Up" - Select previous variable at selected line.
    • "ctrl" + "Page Down" - Select next line.
    • "ctrl" + "Page Up" - Select previous line.
    • "numpad -" - Decrease value of selected variable.
    • "numpad +" - Increase value of selected variable.
    • "shift" + "numpad -" - Decrease value of selected variable, small step.
    • "shift" + "numpad +" - Increase value of selected variable, small step.
    • "ctrl" + "numpad -" - Hide last variable line.
    • "ctrl" + "numpad +" - Show next variable line.

    • "s" - Save all variables for current entity and global ones.
    • "ctrl" + "s" - Save selected entity variable for all entities. Does nothing if global one is selected.

    Ability to see larvae/dollies on mini-map while Dominance is more than 0. You need more than 50 in original.

    While camera is still, there is different rendering for landscape.

    Key to the next inhabited world is gained as soon as key giving line is available. Key giving line is blocked by first greeting at Podish and by ruBeecation at VigBoo.

    Number of bytes of RAM allocated for this game is saved in "memstats.dmp".
